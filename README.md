# 🍽️ Fridgy

A recipe discovery Android app that uses the **Google Gemini AI** to generate personalised recipes from whatever ingredients you have on hand, powered by the **MealDB** API for browsing curated meals by category.

## Features

- 🤖 **AI Recipe Generation** — Type in your ingredients and Gemini generates 3 custom recipes with ingredients, steps, nutrition, and chef tips
- 🗂️ **Category Browsing** — Browse Breakfast, Lunch, Dinner, Dessert, and Salads via MealDB (no API key needed)
- ❤️ **Favourites & Recents** — Recipes are persisted locally so your history is always available
- 🎨 **Themeable UI** — Multiple colour themes available from the profile icon

## Setup

### 1. Clone the repository

```bash
git clone https://github.com/your-username/Fridgy.git
cd Fridgy
```

### 2. Get a Gemini API key

> **Free tier** is sufficient — the app uses `gemini-2.5-flash` with a fallback to `gemini-1.5-flash`.

### 3. Add the API key to `local.properties`

Open (or create) the file `local.properties` in the **root** of the project (the same folder as `build.gradle.kts`). Add the following line:

```properties
gemini.api.key=YOUR_API_KEY_HERE
```

Replace `YOUR_API_KEY_HERE` with the key you copied in step 2.

**Example `local.properties`:**
```properties
## This file is automatically generated by Android Studio.
# Do not modify this file -- YOUR CHANGES WILL BE ERASED!
sdk.dir=C\:\\Users\\YourName\\AppData\\Local\\Android\\Sdk
gemini.api.key=AIzaSy...your-key-here...
```

> ⚠️ `local.properties` is listed in `.gitignore` and will **never** be committed to version control. Never share this file or hard-code your API key in source files.

### 4. Sync and run

1. Open the project in **Android Studio**
2. Wait for Gradle to sync (it reads `gemini.api.key` from `local.properties` at build time and injects it as `BuildConfig.GEMINI_API_KEY`)
3. Select a device or emulator running **API 31+**
4. Click **Run ▶**

## How the API key flows through the build

```
local.properties
    └─ gemini.api.key
           │
           ▼  (read by app/build.gradle.kts)
   buildConfigField "GEMINI_API_KEY"
           │
           ▼  (accessed at runtime)
   BuildConfig.GEMINI_API_KEY
           │
           ▼
   HomeActivity → Gemini REST API
```

## Running without an API key

If `gemini.api.key` is missing or empty, the app falls back to **mock data** — two placeholder recipes are shown whenever you tap **Generate**. All category browsing (MealDB) still works normally as it requires no API key.

## Troubleshooting

| Problem | Solution |
|---------|----------|
| **"Failed to generate" toast** | Check that your API key is correct and has Gemini API access enabled in Google Cloud Console |
| **Blank recipe cards** | The MealDB API is rate-limited; wait a moment and switch tabs |
| **Build error: `gemini.api.key` not found** | Make sure `local.properties` exists at the project root and contains the key |
| **minSdk error on emulator** | Use an AVD running API 31 (Android 12) or higher |
